**MODULE 8 JOURNAL**

Artemis Financial is a financial services company seeking to provide consumers with a variety of
customizable financial plans tailored to individual needs. As part of this initiative
, the company is prioritizing the integration of strong security measures to protect
sensitive data during transmission and prevent unauthorized access.

One aspect of the project I particularly enjoyed was conducting dependency 
checks to identify potential vulnerabilities within the application. This process 
reinforced the importance of writing secure code from the outset. When security is 
embedded into development practices, it significantly reduces the likelihood of 
exploitable weaknesses that attackers could use to compromise the system. Strong software
security not only protects data but also enhances a company’s credibility and overall 
reputation. When customers trust that their information is secure, they are more likely 
to engage with and remain loyal to the company.

A challenge I encountered during the vulnerability assessment process was 
distinguishing between legitimate vulnerabilities and false positives. Accurately
identifying false positives is essential to ensure that time and resources are
focused on real security threats. To strengthen the application’s security posture, 
I implemented an SSL certificate and ensured secure HTTP (HTTPS) communication was
properly configured to encrypt data in transit.
Looking ahead, one strategy I would continue to apply is carefully analyzing 
vulnerability scan results to verify findings and effectively determine appropriate mitigation techniques, particularly by identifying and resolving false positives. To ensure the software remained both functional and secure, I repeatedly ran the application and performed dependency checks throughout the refactoring process. Each time changes were made to the codebase, I retested the project to confirm no new vulnerabilities were introduced.

One of the most valuable resources I utilized was testing and reviewing 
the project through the web browser within Eclipse, which allowed me to monitor 
functionality and security configurations in real time. A key assignment I would
highlight is my Vulnerability Assessment Report, as it demonstrates my ability to
interpret scan results, assess security risks, and understand the implications of the 
findings in a practical development environment.
